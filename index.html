Ruby / RoR

Project: Prepify

Timeline: 2 weeks (1.0) and 2 days (2.0)

Tech Stack: Ruby on Rails, Javascript, AJAX, Jquery, Postgres, Backbone.js, HTML5, CSS3

Description: Working in conjunction with graduate students from the McCombs School of Business, the Prepify team created a functioning model for a quiz client designed to make SAT prep within reach of everyone. The second version was created at a subsequent hackathon as an attempt to reboot the first.

Technical challenge (and how I overcame it): After writing code for a subscription email designed to introduce new users, I was having trouble making Rails ActionMailer fire on the signup event. After tracing the stack of the rails server, I realized the original rails create action was never being called - instead it was a custom Devise function (an authentication gem) that was getting invoked, with user as well as session creation logic. To fix the issue, I introduced a new custom create action that inherited from the Devise function, invoking the same method with the keyword "super" to ensure authentication was preserved, while adding the new mail logic below. Classical inheritance: a programmer's best friend.

Codebase: Original Repo, Repo 2.0
