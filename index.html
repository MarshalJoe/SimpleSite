Ruby / Rails

Project: Prepify

Timeline: 2 weeks (1.0) and 2 days (2.0)

Tech Stack: Ruby on Rails, Javascript, AJAX, Jquery, Postgres, Backbone.js, HTML5, CSS3

Description: Working in conjunction with graduate students from the McCombs School of Business, the Prepify team created a functioning model for a quiz client designed to make SAT prep within reach of everyone. The second version was created at a subsequent hackathon as an attempt to reboot the first.

Technical challenge (and how I overcame it): After writing code for a subscription email designed to introduce new users, I was having trouble making Rails ActionMailer fire on the signup event. After tracing the stack of the rails server, I realized the original rails create action was never being called - instead it was a custom Devise function (an authentication gem) that was getting invoked, with user as well as session creation logic. To fix the issue, I introduced a new custom create action that inherited from the Devise function, invoking the same method with the keyword "super" to ensure authentication was preserved, while adding the new mail logic below. Classical inheritance: a programmer's best friend.

Codebase: Original Repo, Repo 2.0


Javascript

Project: Window Resize Bug

Timeline: 1 day

Tech Stack: Javascript, jQuery, Python, Django

Description: While assisting a Makersquare mentor's freelance project, I was asked to close a simple JS ticket. The bug: every time a user resized the window of the embedded video content we provided (through a python/django app that uploaded it to a CDN), the video would start over. I started by tracing the console logs associated with initializing the video's dimensions. Running through the code, I saw the play function had become coupled to the window resizing. To solve the issue, I refactored the offending play code to accept an extra argument - a boolean that would be passed as true only when the video was being first initialized. Then I wrapped the play code in an if conditional predicating the boolean. A quick patch until refactoring.


HTML5 / CSS3

Project: Contact Form

Timeline: 1 day

Tech Stack: PHP, jQuery, Javascript, AJAX

Description: Absent some serious client-side goodness like Angular, AJAX is the only way to go for a seamless form that doesn't hang on hitting the server.

Technical challenges (and how I overcame them): (1) After successfully writing the code to AJAX the form's results and display loading and success messages, I was having trouble with the form hanging - taking almost until timeout to complete the mail function successfully. After researching, I found that changing the configuration of /etc/hosts to list a FQDN (full qualified domain name) solved the issue swimmingly. (2) Putting the form up on the site, I noticed that I wasn't able to click into the form fields - except halfway through the message text area. I realized it had to be a CSS issue and, inspecting the form's elements, discovered that the fixed position block containing my name and job title, although it was fully transparent, was overlaying the form and preventing selection. Simple fix: z-index! I simply positioned the form element and added a z-index higher than that of the title.
